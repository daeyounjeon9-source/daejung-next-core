DAEJUNGNEXT V3002 AUTH/ADMIN REVIEW PATCH

수정 기준: v3002.zip

완료 항목
1. 로그인/회원가입/판매자/관리자 화면 문구 정리
   - 포스틱/요리사/관리자 대시보드로 이동 이상 문구 검색 및 제거 상태 확인
   - 로그인/회원가입/판매자/관리자 화면의 주요 영문 문구를 한글 중심으로 정리

2. 회원가입 정식 절차 유지
   - 이름, 이메일, 휴대폰번호, 비밀번호, 비밀번호 확인
   - 비밀번호 규칙 문구 표시
   - 필수 동의 4개 체크 후 가입
   - 일반 회원가입은 member 권한만 생성

3. 판매자 회원가입 정식 절차 유지
   - 대표자/담당자명, 이메일, 비밀번호, 비밀번호 확인, 휴대폰/연락처
   - 상호/브랜드명, 사업자등록번호, CS연락처, 정산계좌 등 입력
   - 필수 동의 5개 체크 후 판매자 신청
   - 판매자는 seller 권한으로 생성, 관리자 검수 대기 상태

4. 로그인 검증
   - 아무 이메일/아무 비밀번호 통과 차단
   - 회원가입 안 된 계정 로그인 차단
   - 역할별 로그인 분리: 회원/member, 판매자/seller, 관리자/admin

5. 관리자 지정 구조 추가
   - 일반 회원가입으로 admin 생성 금지
   - 관리자 대시보드 안에 '관리자 지정' 영역 추가
   - 기존 관리자가 가입된 회원 이메일을 입력해야 admin 권한 부여
   - 가입 계정 권한 현황 표시

터미널 검수 결과
- npm install 성공
- npm run build 성공
- npm run dev -- --host 성공
- Local: http://localhost:5173/
- Network: http://172.26.36.32:5173/ (작업 컨테이너 기준)

사용자 PC 휴대폰 확인 절차
1. PC 터미널에서 npm run dev -- --host 실행
2. 다른 터미널에서 ipconfig 실행
3. IPv4 주소 확인
4. 휴대폰 브라우저에서 http://<IPv4>:5173 접속

주의
- 현재 인증은 로컬 테스트 저장소(localStorage) 기반입니다.
- 실제 배포 전 Firebase/Supabase 인증과 서버 권한검증으로 교체해야 합니다.
